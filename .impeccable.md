## Design Context

### Users

Claude Code users — developers managing skills, plans, agents, commands, CLAUDE.md, and settings. They open this home screen to orient quickly and jump into a category or recent file. Context: local desktop/dev environment, often mid-session, switching between coding and config.

### Brand Personality

Calm · elegant · easy to navigate. A beautiful developer tool — quiet confidence, not flashy. Feels intentional and precise, like a well-made IDE surface rather than a marketing dashboard.

### Aesthetic Direction

- Theme: dark Clay system (`DESIGN.md`) — deep teal-black surfaces, warm off-white text, category accent dots.
- Visual tone: refined density with clear hierarchy; typography and space do the work; accents are rare.
- References: calm developer tools with strong navigation hierarchy (not Notion, not generic SaaS admin).
- Anti-references: AI slop (purple glow, identical card grids, hero metrics, nested cards, decorative gradients), generic SaaS dashboards with equal-weight widgets.

### Design Principles

1. **Browse first** — Category browsing is the primary job on home; give it the most visual weight and page real estate.
2. **Calm hierarchy** — One clear primary region, quieter secondary regions (recent + config); never three equal competing cards.
3. **Rhythm over uniformity** — Tight grouping inside lists; generous separation between sections; vary container treatment so not everything is a raised card.
4. **Tokens only** — Colors, radii, and spacing from Clay/`DESIGN.md` and Tailwind theme tokens; no arbitrary decorative chrome.
5. **Elegant density** — Comfortable, not sparse: fill the dead zone with purposeful structure, not empty dashboard panels.
